# Code Review 審查標準

## 專案架構

> 依據 repo 的 `CLAUDE.md` / `AGENTS.md` 中 Repository Guidelines 所定義的專案結構與技術棧。

## 審查項目

### 1. Spec 符合度

- 成功標準是否達成
- 任務 DoD 是否滿足
- 變更是否只覆蓋必要檔案
- 是否遺漏邊界情境

### 2. 架構合規

- 是否遵循 repo 規定的架構模式與分層
- 是否使用 repo 規定的 API 呼叫方式與客戶端工具
- UI 是否使用專案設計系統元件
- 狀態管理是否符合專案慣例
- 後端是否遵循規定的架構分層與 DI 模式
- 錯誤處理是否完整

### 3. 程式碼品質

- 邏輯與邊界正確
- 無硬編碼/魔術數字
- 避免重複造輪子
- 安全性（輸入驗證、敏感資料）
- 效能（避免不必要查詢與 N+1）

### 4. 一致性

- 命名與格式符合專案慣例
- 錯誤處理模式一致

### 5. 測試品質

- 新功能必須有對應測試（缺測試 = P1）
- 測試涵蓋 happy path + error/edge
- Mock 只針對外部依賴
