# /explore — 輕量探索

> 純讀取：不修改任何檔案、不寫 sdd_context、不觸發 SOP 管線。

## 環境資訊

- 當前分支: !`git branch --show-current`

## 輸入

探索目標：$ARGUMENTS

---

## 與其他指令的邊界

| 需求 | 使用 |
|------|------|
| 「這能不能做？大概多大？」 | `/explore`（本指令）|
| 「X 到底怎麼運作？是真的嗎？」 | `/verify`（事實驗證） |
| 「我要做 X 功能」 | `/s0-understand`（SOP） |
| 「X 壞了」 | `/debug` |

> **關鍵區別**：`/explore` 回答「可行嗎？範圍多大？」；`/verify` 回答「X 是真的嗎？」

---

## 探索流程

### 1. 解析探索目標

用 sequential-thinking 拆解：核心是什麼？需確認哪些技術面向？

### 2. 快速定位相關程式碼

Grep/Glob/Read：類似功能現有實作、可能的修改切入點、已知限制與約束

### 3. 估算

- **影響範圍**：預計動到哪些層（DB/Backend/Frontend）
- **粗略規模**：S（<半天）/ M（1-2天）/ L（>2天）
- **主要技術挑戰或風險**

### 4. 建議

- 是否值得啟動 SOP？
- 建議的 work_type
- 待確認的假設

- 純讀取：不修改任何檔案、不寫入任何檔案
- 不建立 sdd_context
- 不觸發 SOP 管線
- DB 僅 SELECT/DESCRIBE/SHOW
